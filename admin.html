            AI assistant — reads your current setup automatically

            Each register, printer, and card reader needs a fixed network address on your store network. Enter the hardware address (MAC) from each device sticker on the equipment record, reserve that address on your router, then paste the router's device list below and click Apply to sync.

            How to set up reserved addresses on the router
            1. In the router settings, note your store network address and subnet mask.
            2. Plug the router in: internet cable to the modem, store devices to LAN ports or store Wi‑Fi.
            3. For each POS device, copy the hardware address (MAC) from the sticker and tell the router to always give it the IP you entered on the equipment record.
            4. Connect registers and card readers to a store-only Wi‑Fi network — not guest Wi‑Fi. Write the Wi‑Fi name and password in Network notes.
            5. Enter the same MAC on each Equipment row. Paste the router's connected-device list → Parse & match → Apply.
            6. Backup internet test: run a sale on normal internet, unplug the modem cable, run a sale again on cellular backup (if the router has it).

            Your router's LAN subnet (often shown as something like 192.168.1.0/24).

            After reserved addresses are set on the router, copy its list of connected devices. Each line should include a name, IP address, and hardware (MAC) address.

            Developer Tools

            How to update the site

            "Updating the site" means taking changes that work on your test computer and pushing them to the live site so real visitors see them. Always test locally first, then follow the steps below.

            Always back up first. Before every update, click "Download SQL backup" below and save the file. If anything goes wrong, you can restore from it.

            1. Decide what kind of update it is

            Frontend only

            Text, images, prices, page styling, popups.
            → Update files, then hard-refresh. No migration needed.

            Backend logic

            Changes inside the backend/ folder.
            → Update files + restart the server.

            Database change

            Anything that mentions a migration or "adds a column/table."
            → Update files + run migrations + restart.

            2. Follow the steps in order

            1. 1
              Back up. Click "Download SQL backup" below and save the file somewhere safe.
            2. 2
              Get the new files onto the server. On the server, pull the latest code: git pull (this is the SSH/server step your developer usually runs).
            3. 3
              Install new libraries (only if the update added any): npm install
            4. 4
              Run migrations (database updates only) — use the "Run pending migrations" button below and type RUN MIGRATIONS when asked. Skip for frontend-only updates.
            5. 5
              Restart the backend (backend or database updates): pm2 restart hmherbs-backend
            6. 6
              Check the live site. Hard-refresh with Ctrl + F5 and click through whatever changed to confirm it works.
            If something breaks: restore the backup you downloaded in step 1, and check the server logs with pm2 logs hmherbs-backend (or send them to your developer).
